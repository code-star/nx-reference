# ADR-0006: Standalone components and signal-based Angular APIs

> **date:** 2026-07-03\
> **status:** accepted

## context

The legacy code is NgModule-first Angular 12: `@NgModule` declarations (`UiModule`,
`SharedServicesModule`, `RemoteEntryModule`, `AppModule`), constructor DI, `@Input()` decorators,
`*ngIf`/`*ngFor` structural directives, `platformBrowserDynamic().bootstrapModule()`, and
`HttpClientModule`. Angular 21 makes **standalone** the default and provides the **signals** input
API, `inject()`, the built-in control flow block syntax, and `provide*` bootstrap functions. The
rebuild should use "the latest Angular way of working" while keeping the UI design and public
contracts identical.

## decision

Author all new Angular code standalone and signal-first:

- **Standalone components/directives/pipes** (`standalone: true` default) — no NgModules in new code.
- **Bootstrap:** `bootstrapApplication(App, { providers: [provideRouter(...), provideHttpClient()] })`.
- **DI:** `inject()` instead of constructor injection.
- **Inputs:** signal inputs via `input()` / `input.required()`, preserving the existing input **names
  and types** (`disabled`, `loading`, `type`, `item`, `rates`, `title`).
- **Control flow:** `@if` / `@for` / `@switch` instead of `*ngIf` / `*ngFor` / `ngSwitch`.
- **HTTP:** `provideHttpClient()` (replaces `HttpClientModule`).
- **Selectors, templates (rendered output), and SCSS remain identical** to preserve the visual design
  (see `docs/design/components.md`).
- MF host/remote use standalone bootstrap and `provideRouter` with lazy `loadChildren`/`loadComponent`
  as generated by the Nx MF generators.
- **Zoneless change detection** via `provideZonelessChangeDetection()` — the Angular 21.2 / Nx 23
  scaffolded default (no `zone.js`). Reactive state that must trigger change detection is held in
  signals; RxJS subscriptions update signals to refresh the view.

## alternatives considered

- **Keep NgModule-first Angular:** works on Angular 21 but contradicts "latest Angular way of
  working". Rejected.
- **Partial adoption (standalone but constructor DI / old control flow):** inconsistent and misses the
  modernisation intent. Rejected in favour of full adoption.
- **Keeping zone.js change detection:** rejected. Nx 23 / Angular 21.2 **scaffolds zoneless by
  default** — `zone.js` is not installed and the generated Jest `test-setup.ts` uses
  `setupZonelessTestEnv`. Re-adding zone.js would fight the toolchain default. We therefore **adopt
  zoneless change detection** (`provideZonelessChangeDetection()`), which is consistent with the
  signal-first component design.

## rationale

- Standalone + signals is the documented default and idiomatic surface for Angular 21 and integrates
  cleanly with Nx's standalone MF generators.
- Preserving input names/types and template output keeps design and consumer contracts intact while
  modernising the implementation.

## consequences

- **Easier:** less boilerplate; tree-shakable providers; direct interop with Nx standalone MF; no
  zone.js payload.
- **Harder:** stories, specs, and app wiring must be rewritten for standalone (import components
  directly, no `TestBed` module declarations); signal inputs change how tests set inputs
  (`fixture.componentRef.setInput` / `componentInstance.x` accessors); zoneless requires host
  component state (e.g. `rates`, `loading`) to be signals so the view updates. Handled in Phases 3–8.
- Zoneless change detection is **adopted** as the scaffolded default (amended from the original
  "deferred" position after Phase 1 toolchain verification).
